>_ HOW PASSWORD STRENGTH DETECTORS ACTUALLY WORK

Have you ever wondered how a password strength checker gives you a score? It's not magic — it's math, entropy, and a few simple rules that security experts have refined over decades.

📊 The Scoring System

Most password strength checkers (including this one) use a points-based system. Each positive attribute adds points:

  • Length: 8+ characters = +20 points, 12+ = +10 more, 16+ = +10 more
  • Lowercase letters: +15 points
  • Uppercase letters: +15 points
  • Numbers: +15 points
  • Special characters (!@#$%): +15 points
  • Common passwords: -20 points (penalty)

Then the total is capped at 100, and a label is applied based on the score.

🔢 What Is Password Entropy?

Entropy measures how unpredictable a password is. It's calculated as:

Entropy = log₂(possible_characters ^ password_length)

Example:

  • "password" (8 lowercase letters) → 8²⁶ possibilities → Very low entropy
  • "P@ssw0rd123!" (12 chars, mix) → 12⁹⁴ possibilities → Very high entropy

⏱️ How Crack Time Is Estimated

Crack time is calculated by estimating how many guesses per second a hacker's computer can make:

  • Weak password (0-30 score): Instantly — hackers already have it in their dictionary
  • Weak (30-50): Minutes to hours
  • Moderate (50-70): Days to weeks
  • Strong (70-90): Years
  • Very Strong (90-100): Centuries

🧠 What Hackers Actually Do

Real hackers don't "guess" one password at a time. They use:

  • Dictionary attacks: Trying millions of leaked passwords
  • Brute force: Trying every possible combination
  • Hybrid attacks: Adding numbers to dictionary words (like "password123")
  • Mask attacks: Trying patterns like "Name1990"

✅ Is This Tool Accurate?

Yes — for estimating complexity. But real-world security depends on more than just password strength. Always use unique passwords and enable 2FA.
